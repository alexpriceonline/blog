The boilerplate is a barebones HTML and SCSS starter-kit. It makes starting simple front-end projects a breeze and will have you up and coding in minutes.

View on Github

So what's included?

src is where all of the un-processed files should live (SCSS, uncompressed JS or Coffeescript etc.). The boilerplate is Javascript processing independent. 

src
└── scss
    ├── base
    │   ├── _colors.scss
    │   └── _global.scss
    ├── mixins
    │   ├── _break.scss
    │   ├── _clearfix.scss
    │   ├── _font-size.scss
    │   └── _rem.scss
    ├── modules
    ├── vendor
    │   └── _normalize.scss
    ├── _variables.scss
    └── styles.scss

Directory walkthrough

base/

Put global stuff here. _grid.scss would be a good example.

  • _colors.scss is for colour variable declarations.
  • _global.scss is for site-wide styles (global box-sizing etc.).

mixins/

SCSS mixins go here—duh. Mixins currently included are:

break : a simple responsive breakpoint mixin.
The boilerplate is mobile-first, code inside break($size) blocks are applied when the browser is larger than $size.

In this example small is being used, meaning the display: inline-block; and float: left; properties are applied to .footer-logo-link whenever the browser width is larger than the values of $small (declared in _variables.scss). 

.footer-logo-link {
    display: block;
    height: rem(55px);

    @include break(small) {
        display: inline-block;
        float: left;
    }
}
clearfix : a simple float clearing mixin.

Including @import clearfix(); in a style block will stop it collapsing if elements inside are floated.

font-size : a rem based font-size fallback mixin.

Some browsers do not support rem based font sizes—Damn you IE9-10. This mixin simply provides both the rem version and a px fallback in one line. 

.footer-project-title {
    @include fontSize(14px);
    ...
}

// Outputs
.footer-project-title {
    font-size: 14px;
    font-size: rem(14px); // rem calculation, see below.
    ...
}
rem : a px to rem conversion mixin.

The rem mixin is the most complecated, but most useful mixin in the boilerplate.

It saves you bags of time by letting you write all CSS sizes in your beloved px. The SCSS processor converts them all into the rem equivelent (based on the $rem-base variable).

It accepts multiple inputs and although it isn’t required, I always write the px suffix to keep things explicit.

Padding is a great example (assuming $gutter: 18px and $rem-base: 16px): 

padding: rem(0 $gutter);

// Outputs
padding: 0 1.125rem;

modules/

Put your self-contained components here.

vendor/

The vendor directory is where you should put all of your 3rd party style sheets. The boilerplate comes with Normalize.

_variables.scss

This is where you should declare your basic variables. Things like $rem-base size and responsive break point values.

_styles.scss

This is the main import file. All of the above should be referenced here; seperated by folder and sorted alphabetically.

Sometimes you may find file imports are dependant on other files and need to be places below them (out of alphabetical order). This is fine, just place a comment after the import stating the dependancy.

Gulp

I'm not a massive fan of front-end build systems. But with open source projects it is important to have consistancy between developers. Gulp is super simple and can be extended easiliy on a project to project basis.

The Gulp setup that comes with this boilerplate is soley for SCSS processing and improses no Javascript opionion on the developer. That being said, my personal preference is to keep code as vanilla as possible; #NoCoffeeScriptHere.

And that's it

It's as simple as that. This a super fast and lightweight way to kick off a new front-end project. But wait! I need your help to make the boilerplate even better. Please consider contributing to the project. Every commit, comment and share will help to make this the best boilerplate possible.
